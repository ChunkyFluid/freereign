Container queries vs. media queries

Media queries respond to the viewport. That's a problem for reusable components: a card in a wide main column and the same card in a narrow sidebar both see the same viewport, so they can't lay themselves out differently. Container queries fix this — a component responds to the size of its own container, wherever it's placed. It's the missing piece for truly component-driven responsive design.

The two-step setup

First, declare an element as a query container. Then query it from its descendants.

/* 1. Mark the parent as a container */
.card-wrapper {
  container-type: inline-size;   /* query its width */
  container-name: card;          /* optional name */
}

/* 2. Style children based on the container's width */
@container card (min-width: 400px) {
  .card { display: grid; grid-template-columns: 120px 1fr; }
}
  • container-type: inline-size — the most common; queries the container's inline (horizontal) size.
  • container-name — optional label so @container name (...) targets a specific ancestor.
  • @container — works just like @media, but measures the nearest matching container instead of the viewport.

A worked example

One card component, dropped into two different columns, adapting itself to each:

.card-wrapper { container-type: inline-size; }

.card { display: flex; flex-direction: column; }     /* narrow: stacked */

@container (min-width: 380px) {
  .card { flex-direction: row; align-items: center; } /* wide: side by side */
}

Drop .card-wrapper in a 300px sidebar and the card stacks; drop it in a 700px main column and it goes horizontal. No viewport breakpoints, no component variants.

Container query units

Container queries bring their own relative units, sized against the query container instead of the viewport:

  • cqw — 1% of the container's width
  • cqh — 1% of the container's height
  • cqi — 1% of the container's inline size (usually width)
  • cqb — 1% of the container's block size
.card__title { font-size: clamp(1rem, 5cqi, 1.5rem); } /* fluid within the container */

Browser support & Tailwind

Container queries are supported in all major browsers (Chrome, Edge, Safari, Firefox) since 2023 — safe for production today. In Tailwind, the first-party @tailwindcss/container-queries plugin adds @container and @md:-style variants:

<div class="@container">
  <div class="flex flex-col @md:flex-row">...</div>
</div>

Gotchas

  • An element can't query itself. You need a wrapper: the container is the parent, the queried styles target its descendants.
  • Containment has side effects. container-type: size (querying height) requires the container's block size to be defined, or content can collapse — inline-size is safer and what you want 90% of the time.
  • Name your containers when you nest them, so a query targets the right ancestor.
