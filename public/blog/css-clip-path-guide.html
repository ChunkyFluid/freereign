CSS clip-path: The Complete Guide

The clip-path property crops an element to any shape — circles, hexagons, arrows, stars — without images or SVG masks. What is clip-path?

clip-path defines a region of an element to keep — everything outside that region becomes invisible (but still occupies layout space unless you handle that separately). Unlike overflow: hidden, which only clips to a rectangle, clip-path clips to any shape: basic geometric functions or arbitrary polygons. It's hardware-accelerated and fully animatable.

The basic shape functions

/* Circle — radius + center position */
clip-path: circle(50% at 50% 50%);

/* Ellipse — x-radius y-radius at center */
clip-path: ellipse(40% 50% at 50% 50%);

/* Inset — crop in from each edge (top right bottom left), optional round */
clip-path: inset(10% 20% 10% 20% round 12px);

/* Polygon — the powerful one: a list of x y points */
clip-path: polygon(50% 0%, 100% 100%, 0% 100%); /* triangle */
  • circle() — one radius and a center point. Great for avatars and reveal effects.
  • ellipse() — two radii (x and y) and a center.
  • inset() — rectangle cropped inward from each edge; round adds corner radius.
  • polygon() — arbitrary shapes from a list of coordinates. This is where everything interesting happens.

The polygon coordinate system (the part everyone gets stuck on)

polygon() takes a comma-separated list of x y point pairs. The browser connects them in order and fills the enclosed region. Coordinates are measured from the top-left corner of the element:

  • 0% 0% → top-left
  • 100% 0% → top-right
  • 100% 100% → bottom-right
  • 0% 100% → bottom-left

The first value in each pair is horizontal (relative to width), the second is vertical (relative to height). Because they're percentages, the shape scales with the element — make the box bigger and the hexagon stays a hexagon. Walk the points clockwise and you'll trace the outline:

/* A flat-top hexagon: six points, clockwise from top-left */
clip-path: polygon(
  25% 0%,    /* top-left corner   */
  75% 0%,    /* top-right corner  */
  100% 50%,  /* right point       */
  75% 100%,  /* bottom-right      */
  25% 100%,  /* bottom-left       */
  0% 50%     /* left point        */
);

Copy-paste shape recipes

/* Triangle */
clip-path: polygon(50% 0%, 0% 100%, 100% 100%);

/* Hexagon */
clip-path: polygon(25% 0%, 75% 0%, 100% 50%, 75% 100%, 25% 100%, 0% 50%);

/* Arrow (pointing right) */
clip-path: polygon(0% 20%, 60% 20%, 60% 0%, 100% 50%, 60% 100%, 60% 80%, 0% 80%);

/* Chevron */
clip-path: polygon(75% 0%, 100% 50%, 75% 100%, 0% 100%, 25% 50%, 0% 0%);

/* 5-point star */
clip-path: polygon(50% 0%, 61% 35%, 98% 35%, 68% 57%, 79% 91%, 50% 70%, 21% 91%, 32% 57%, 2% 35%, 39% 35%);

/* Diagonal section divider */
clip-path: polygon(0 0, 100% 0, 100% 85%, 0 100%);

Animating clip-path

clip-path interpolates smoothly as long as both shapes use the same function and the same number of points. This is the trick behind hover reveals and morphing buttons:

.reveal {
  clip-path: polygon(0 0, 0 0, 0 100%, 0 100%); /* collapsed at left */
  transition: clip-path 400ms ease;
}
.reveal:hover {
  clip-path: polygon(0 0, 100% 0, 100% 100%, 0 100%); /* full box */
}
Animating between different functions (e.g. circle() → polygon()) is not interpolated and will snap. Keep the same function and matching point count for smooth motion.

Browser support & the prefix

Basic shapes are supported across Chrome, Edge, Firefox and Safari. Safari historically required -webkit-clip-path, so declare both for safety:

.shape {
  -webkit-clip-path: polygon(50% 0%, 0% 100%, 100% 100%);
  clip-path: polygon(50% 0%, 0% 100%, 100% 100%);
}

clip-path in Tailwind CSS

Tailwind has no built-in shape utilities, but arbitrary values work:

<div class="[clip-path:polygon(50%_0%,0%_100%,100%_100%)]">
  Triangle
</div>

Underscores stand in for spaces inside the arbitrary value.

Gotchas

  • Layout still reserves the full box. Clipped-away pixels are invisible but the element's original rectangle still affects layout — neighbours don't reflow to the shape.
  • Mind the click area. Pointer events follow the clipped shape in most browsers, so corners you cropped away won't be clickable.
  • Use percentages for responsiveness. Pixel coordinates won't scale; percentages keep the shape proportional at any size.

FAQ

How does the polygon coordinate system work?

Each point is an x y pair measured from the top-left corner, as a percentage of the element's width and height. Connect the points in order to trace the visible shape.

Can you animate clip-path?

Yes — between shapes that use the same function and the same number of points. That's how reveal and morph effects work.

Is clip-path supported everywhere?

All modern browsers support the basic shapes. Add the -webkit-clip-path prefix for older Safari.

How do I make a hexagon?

clip-path: polygon(25% 0%, 75% 0%, 100% 50%, 75% 100%, 25% 100%, 0% 50%) — six points tracing a flat-top hexagon.
