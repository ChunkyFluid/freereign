CSS Fluid Typography with clamp()

What is fluid typography?

Fluid typography makes font sizes scale smoothly with the viewport instead of jumping at fixed breakpoints. Rather than declaring font-size: 2rem at mobile and 3rem at desktop with a media query in between, you describe a range and let the browser interpolate every size in between. One line replaces a stack of breakpoints.

The clamp() function

Fluid type is built on CSS clamp(), which takes three values:

font-size: clamp(MIN, PREFERRED, MAX);
  • MIN — the smallest the value is ever allowed to get.
  • PREFERRED — the "ideal" value, usually viewport-relative, that the browser uses while it sits between the bounds.
  • MAX — the largest the value is ever allowed to get.

The browser picks the middle value, but never lets it fall below MIN or rise above MAX. That's the whole trick.

The standard recipe

h1 {
  /* min 1.5rem, scales with the viewport, capped at 4rem */
  font-size: clamp(1.5rem, 1rem + 4vw, 4rem);
}

Notice the preferred value is 1rem + 4vw, not just 4vw. That rem part matters — see the accessibility section below.

Why the preferred value mixes rem and vw

A pure viewport unit like font-size: 5vw looks fluid but breaks accessibility: it doesn't respond to browser zoom or the user's font-size preference, because vw is tied to the viewport, not text settings. Mixing in a rem base restores that:

/* ❌ Inaccessible — ignores zoom */
font-size: clamp(1.5rem, 5vw, 4rem);

/* ✅ Accessible — the rem term still responds to zoom */
font-size: clamp(1.5rem, 1rem + 4vw, 4rem);
Rule of thumb: always include a rem component in the preferred value. A common safe split is roughly 1rem + Nvw.

Calculating the slope (the math, briefly)

To make text go from a min size at one viewport width to a max size at another exactly, you solve for the vw coefficient (the "slope") and the rem intercept:

slope     = (maxFont - minFont) / (maxWidth - minWidth)
vw value  = slope * 100
rem value = minFont - slope * minWidth

/* e.g. 16px → 64px between 320px and 1280px viewports */
font-size: clamp(1rem, 0.17rem + 4.17vw, 4rem);

A fluid type scale

Apply the same idea to every step of your type scale and the whole page breathes with the viewport — no breakpoints anywhere:

:root {
  --step--1: clamp(0.83rem, 0.78rem + 0.27vw, 1rem);
  --step-0:  clamp(1rem,   0.91rem + 0.43vw, 1.25rem);
  --step-1:  clamp(1.2rem, 1.07rem + 0.65vw, 1.56rem);
  --step-2:  clamp(1.44rem, 1.26rem + 0.93vw, 1.95rem);
  --step-3:  clamp(1.73rem, 1.48rem + 1.29vw, 2.44rem);
}

Browser support & Tailwind

clamp() is supported in every modern browser and has been for years — no prefix needed. In Tailwind, use an arbitrary value:

<h1 class="text-[clamp(1.5rem,1rem+4vw,4rem)]">Fluid heading</h1>

Gotchas

  • Always cap with MAX. Without it, text keeps growing on ultra-wide monitors until it's comical.
  • Keep MIN readable. Don't let body text clamp below ~16px on phones.
  • Test zoom to 200%. If the text doesn't grow, your preferred value is missing its rem term.
  • Use it for spacing too. clamp() works for padding, margins, and gaps — fluid spacing pairs beautifully with fluid type.

FAQ

What is the clamp() function in CSS?

clamp(MIN, PREFERRED, MAX) returns the preferred value but never lets it drop below MIN or rise above MAX. For typography the preferred value is viewport-relative, so the font size scales fluidly between the two bounds.

Is fluid typography bad for accessibility?

Only if you use a pure viewport unit like 5vw, which ignores browser zoom. Including a rem component in the preferred value — for example 1rem + 4vw — keeps the text responsive to zoom and user font-size settings.

Do I still need media queries with clamp()?

For font sizing, usually not — clamp() interpolates every size between your min and max automatically. You may still use media queries for layout changes, but the typography scales on its own.

Is clamp() supported in all browsers?

Yes. clamp() has been supported in all modern browsers for years and needs no prefix.
